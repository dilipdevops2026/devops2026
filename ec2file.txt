### **Step 1: Open PuTTYgen**

* Run **PuTTY Key Generator**
* Click **Load**
* Select your `.pem` file
  *(If you don‚Äôt see it, change file type to **All Files (*.*)**)*

‚úîÔ∏è You already did this ‚Äî your screen shows **‚Äúimported-openssh-key‚Äù**

---

### **Step 2: (Optional but Recommended) Set a Passphrase**

* Enter a **Key passphrase**
* Confirm the passphrase

> This protects your `.ppk` file if someone copies it

---

### **Step 3: Save as `.ppk`**

* Click **Save private key**
* Click **Yes** if warned about no passphrase
* Save the file as:

  ```
  my-key.ppk
  ```

üéâ **That‚Äôs it! Your `.ppk` file is ready**

## üîê Where Each Key Is Used

| Key Type | Used For                  |
| -------- | ------------------------- |
| `.pem`   | Linux / OpenSSH / AWS CLI |
| `.ppk`   | PuTTY / WinSCP (Windows)  |

---

## üß™ Test the `.ppk` in PuTTY

1. Open **PuTTY**
2. Go to **Connection ‚Üí SSH ‚Üí Auth**
3. Browse and select your `.ppk`
4. Enter hostname & username
5. Click **Open**

---

## ‚ùó Common Mistakes to Avoid

* ‚ùå Clicking **Save public key** (wrong one)
* ‚ùå Renaming `.pem` to `.ppk` manually
* ‚ùå Forgetting username (e.g., `ec2-user`, `ubuntu`)
